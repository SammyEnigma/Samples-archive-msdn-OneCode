Embed language code in URL of an ASP.NET webpage (CSASPNETEmbedLanguageInUrl)

Introduction

The project illustrates how to embed the language code in URL such as http://domain/en-us/ShowMe.aspx. The page will display different content according to the language code. The sample uses url-routing and resource files to localize the content of web page.

Running the Sample

Step 1:   Open the CSASPNETEmbedLanguageInUrl.sln.

Step 2:   Expand the CSASPNETEmbedLanguageInUrl web application and press Ctrl + F5 to show the Default.aspx.

Step 3:   You will see a normal English web page, you can try to modify the value of address bar of browsers, for example, update the "en-us" to "zh-cn".

Step 4:   If you update the url correctly, you will see a Chinese language version of this web page.

Step 5:   Good, if you input a language we don't support in this web application, you will see a default language web page with English.

Step 6:   Validation is finished.

Using the Code

Step 1:   Create a C# "ASP.NET Empty Web Application" in Visual Studio 2012 or Visual Web Developer 2012. Name it as "CSASPNETEmbedLanguageInUrl".

Step 2:    Add one folder, "XmlFolder". In order to display different language content in this web page, we need a database or an xml file to store our data, in this code-sample we just simply add a Language.xml file.

Step 3:   Add three web forms in application's root directory. "ShowMe.aspx" page is for displaying to customers, "InvalidPage.aspx" is for handling error http request, "Start.aspx" is for start url routing.

Step 4:   Add three class files, "BasePage.cs" class is for checking the request url language part and name part, and setting the page's Culture and UICultrue properties. "UrlRoutingHandler.cs" class is for checking the file name and jumping to the InvalidPage.aspx page if the file does not exist. "XmlLoad.cs" class is for loading the xml data and displaying them in ShowMe.aspx page.
[Note]If you want to create more web pages include multiple languages, please inherit the BasePage.cs class for setting page's Cultrue and UICulture.

After register url routes, we need to create a UrlRoutingHandlers to check the request url. The UrlRoutingHanders code like this:

C#

csharp 
/// <summary>
              /// Create this RoutingHandler to check the HttpRequest and
              /// return correct url path.
              /// </summary>
              /// <param name="context"></param>
              /// <returns></returns>
              public IHttpHandler GetHttpHandler(RequestContext context)
              {
                  string language = context.RouteData.Values["language"].ToString().ToLower();
                  string pageName = context.RouteData.Values["pageName"].ToString();
                  if (pageName == "ShowMe.aspx")
                  {
                      return BuildManager.CreateInstanceFromVirtualPath("~/ShowMe.aspx", typeof(Page)) as Page;
                  }
                  else
                  {
                      return BuildManager.CreateInstanceFromVirtualPath("~/InvalidPage.aspx", typeof(Page)) as Page;
                  }
              }

                     string language = RouteData.Values["language"].ToString().ToLower();
                     string pageName = RouteData.Values["pageName"].ToString();
