Work with MediaElement

<o:p> </o:p>

Introduction

<o:p> </o:p>

MediaElement is an essential control to create a media player in Silverlight. It supports playing, pausing, stopping audio/video as well as a lot of interesting features. You can simply control MediaElement by its properties, events and methods.<o:p></o:p>

The following points are covered in this sample:<o:p></o:p>

<![if !supportLists]>·         <![endif]>How to get the total duration of audio/video<o:p></o:p>

<![if !supportLists]>·         <![endif]>How to create a progress bar for MediaElement<o:p></o:p>

<![if !supportLists]>·         <![endif]>Volume control<o:p></o:p>

<![if !supportLists]>·         <![endif]>Play/Pause/Stop audio/video<o:p></o:p>

<![if !supportLists]>·         <![endif]>Events of MediaElement (MarkerReached, MediaEnded, MediaOpened, MediaFailed, CurrentStateChanged)<o:p></o:p>

<![if !supportLists]>·         <![endif]>How to handle MediaElement in full-screen mode<o:p></o:p>

How to get the total duration of Audio/Video

<o:p> </o:p>

You can use the following properties of MediaElement to get the total duration of audio/video that MediaElement plays:<o:p></o:p>

MediaElement.NaturalDuration.TimeSpan<o:p></o:p>

One note is, you can only get the total duration after MediaOpended event.<o:p></o:p>

<o:p> </o:p>

How to create a progress bar for MediaElement

<o:p> </o:p>

We can use Slider control to act as the progress bar of our player. In this sample, the idea to do this is as below. <o:p></o:p>

<![if !supportLists]>·         <![endif]>Initialize Slider.Maximum in MediaOpended event.<o:p></o:p>

<![if !supportLists]>·         <![endif]>Use a DispatcherTimer to syncronize the current position of the video with the Value properties of Slider control<o:p></o:p>

<![if !supportLists]>·         <![endif]>Customize the style of Silder control, in which way we can catch DragStarted/DragCompleted event of the Thumb control. We can then change the position of MediaElement in these two event handlers.<o:p></o:p>

<o:p> </o:p>

Volume control

<o:p> </o:p>

In this sample we use another Slider control to control the volume of MediaElement. Only one line of code is needed to achieve this:<o:p></o:p>

this.MyMediaElement.Volume=this.VolumeSlider.Value;<o:p></o:p>

The above code is put in the ValueChanged event handler of Slider control.<o:p></o:p>

<o:p> </o:p>

Play/Pause/Stop audio/video

<o:p> </o:p>

We can play/pause/stop audio/video by calling MediaElement.Play()/MediaElement.Pause()/MediaElement.Stop() methods<o:p></o:p>

<o:p> </o:p>

Events of MediaElement

<o:p> </o:p>

MarkerReached<o:p></o:p>

This event fires when a marker of the MediaElement is reached. <o:p></o:p>

After markers are loaded from the video and assigned to the Markers properties of MediaElement, we can add our own markers. If we want to do so, a proper place is MediaOpened event.<o:p></o:p>

<o:p> </o:p>

Code to add markers for MediaElement:<o:p></o:p>

this.MyMediaElement.Markers.Add(new TimelineMarker()<o:p></o:p>

          {<o:p></o:p>

              Text = "This marker is added by code!",<o:p></o:p>

              Time = TimeSpan.FromSeconds(8),<o:p></o:p>

              Type = "MyMarker"<o:p></o:p>

          });<o:p></o:p>

To learn how to add markers for a video in Expression Encoder, please refer to:<o:p></o:p>

http://expression.microsoft.com/en-us/cc482957.aspx<o:p></o:p>

<o:p> </o:p>

MediaEnded<o:p></o:p>

This event fires when the media has ended. Common usage of this event is to reset state of certain controls or replay audio/video.<o:p></o:p>

<o:p> </o:p>

MediaOpened<o:p></o:p>

This event fires when the media stream has been validated and opened, and the file headers have been read.<o:p></o:p>

As described in MSDN, for this event we must take care of following things:<o:p></o:p>

Properties in this second category will not contain valid values until the MediaElement raises the MediaOpened event. The following is a list of MediaElement properties that will not be valid until after MediaOpened is raised by that element.<o:p></o:p>

<![if !supportLists]>·         <![endif]>Attributes<o:p></o:p>

<![if !supportLists]>·         <![endif]>AudioStreamCount<o:p></o:p>

<![if !supportLists]>·         <![endif]>AudioStreamIndex<o:p></o:p>

<![if !supportLists]>·         <![endif]>BufferingProgress<o:p></o:p>

<![if !supportLists]>·         <![endif]>DownloadProgress (MediaElement)<o:p></o:p>

<![if !supportLists]>·         <![endif]>Markers<o:p></o:p>

<![if !supportLists]>·         <![endif]>NaturalDuration<o:p></o:p>

<![if !supportLists]>·         <![endif]>NaturalVideoHeight<o:p></o:p>

<![if !supportLists]>·         <![endif]>NaturalVideoWidth<o:p></o:p>

In addition, the Position property will be null before the MediaOpened event. However, Position is different from the other properties in the second group because it is settable. You should not attempt to set Position until MediaOpened is raised. You also should not set an initial Position value in XAML.<o:p></o:p>

<o:p> </o:p>

MediaFailed<o:p></o:p>

This event can occur in the following conditions: <o:p></o:p>

<![if !supportLists]>·         <![endif]>File not found.<o:p></o:p>

<![if !supportLists]>·         <![endif]>Invalid (unrecognized or unsupported) media format.<o:p></o:p>

<![if !supportLists]>·         <![endif]>Unknown media error during playback.<o:p></o:p>

You may see this event fires when you debug your Silverlight application in visual studio. The URL of the video is valid. You can open the video in Windows Media Player.<o:p></o:p>

To solve this issue, the first thing you need to check is whether the URL is a cross-scheme one. Common cause is, the URL of the page is of FILE scheme (e.g. “C:\\...”) and the URL of the video is of HTTP scheme (e.g. http://..). Vice versa.<o:p></o:p>

For more details about this restriction, please check out:<o:p></o:p>

http://msdn.microsoft.com/en-us/library/cc189008(VS.95).aspx<o:p></o:p>

<o:p> </o:p>

CurrentStateChanged<o:p></o:p>

This event fires when the value of the CurrentState property changes. In the sample we catch some state change events and display current state on StatusTextBlock.<o:p></o:p>

How to handle MediaElement in full-screen mode

<o:p> </o:p>

A common requirement regarding MediaElement in full-screen mode is, we expect only the MediaElement to appear in the full-screen mode. This sample demonstrates how to do that. Basically the idea is to enlarge width/height of MediaElement and hide other controls when Silverlight is in full-screen mode. You can prepare another set of UIs to control MediaElement in full-screen mode. <o:p></o:p>

The code regarding full-screen mode can also be found in our samples CSSL3FullScreen/ VBSL3FullScreen.<o:p></o:p>
